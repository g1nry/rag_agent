# RAG RedTeam Agent v0.3.0

**Полноценный агент с RAG + опасными инструментами** для исследования уязвимостей LLM.

Проект позволяет запускать агента, который может:
- Искать информацию в загруженных документах (RAG)
- Выполнять shell-команды
- Читать, писать, удалять файлы
- Менять права доступа
- Создавать директории

## Что это такое

Это **бэкенд-сервис**, который запускается отдельно от вашего основного приложения. Он предоставляет HTTP API, через которое можно общаться с мощным агентом на базе LLM (Qwen2.5 / Llama3.2 и т.д.).

Агент может выполнять как обычные запросы, так и **опасные действия** (выполнение команд в системе, работа с файлами и правами). Поэтому проект предназначен в первую очередь для **исследования уязвимостей LLM** и red teaming.

---

## Возможности

- **ReAct-агент** на LangGraph (планирование + выполнение инструментов)
- **RAG** — поиск по вашим документам
- **6 опасных инструментов**:
  - `shell_execute` — выполнение любых shell-команд
  - `file_read` — чтение файлов
  - `file_write` — запись в файлы
  - `file_delete` — удаление файлов и папок
  - `mkdir` — создание директорий
  - `chmod` — изменение прав доступа
- Поддержка **Human-in-the-Loop** (подтверждение опасных действий)
- Два формата API: `/api/v1/chat` (совместимость) и `/api/agent/chat` (новый)

## Требования

- Python 3.11+
- Ollama (локально или по сети)
- Модель **Qwen2.5:7b** (рекомендуется) или **llama3.2:3b**
- 8+ ГБ RAM (для 7B модели)

## Быстрый старт

```bash
# 1. Клонируй репозиторий
git clone https://github.com/g1nry/rag_agent.git
cd rag_agent

# 2. Создай виртуальное окружение
python3 -m venv .venv
source .venv/bin/activate

# 3. Установи зависимости
pip install -e .

# 4. Запусти Ollama и скачай модель
ollama serve
ollama pull qwen2.5:7b

# 5. Запусти бэкенд
python -m rag_agent
```

Открой в браузере: http://localhost:8000

## Примеры использования

### 1. Простой запрос

```bash

  -H "Content-Type: application/json" \
  -d '{
```

## Конфигурация (`config.toml`)

Самое важное — секция `[ollama]`:

```toml
[ollama]
base_url = "http://localhost:11434"
chat_model = "qwen2.5:7b"
embedding_model = "nomic-embed-text"
timeout = 120.0
```

**Советы:**
- Если Ollama в Docker — используй `http://host.docker.internal:11434`
- Увеличивай `timeout`, если модель медленная
- `temperature` лучше ставить 0.2–0.4 для стабильности

## Возможные ошибки и решения

### 1. `ImportError: cannot import name 'retrieval_service'`

**Решение:** Убедись, что в `src/rag_agent/services/retrieval_service.py` в самом конце есть строка:

```python
retrieval_service = RetrievalService(...)
```

### 2. Модель отвечает на китайском/английском

**Решение:**
- Используй модель `qwen2.5:7b` (лучше всего понимает русский)
- Уменьши `temperature` до 0.2–0.3
- Перезапусти сервис

### 3. Агент "не видит" файлы, которые только что создал

**Решение:** Это известная проблема слабых моделей. Используй `qwen2.5:7b` или `llama3.1:8b`. На 3B моделях это происходит часто.

### 4. `utf-8 codec can't decode byte`

**Решение:** Ошибка в выводе инструментов. Исправлено в версии 0.3.0. Обнови код.

### 5. Ollama недоступна (`503`)

**Решение:**
- Убедись, что `ollama serve` запущен
- Проверь `base_url` в `config.toml`
- Если в Docker — используй `host.docker.internal`

### 6. Порт 8000 занят

**Решение:**
```bash
lsof -i :8000
kill -9 <PID>
```

Или поменяй порт в `config.toml` (`app.port = 8001`)
